Create `src/lib/hooks/use-recurring.ts` with ALL recurring TanStack Query hooks following the established pattern from use-subscriptions.ts:

**Query key factory:**
```typescript
export const recurringKeys = {
  all: ["recurring"] as const,
  dashboard: () => [...recurringKeys.all, "dashboard"] as const,
  masters: () => [...recurringKeys.all, "masters"] as const,
  masterList: (filters: RecurringMasterFilters) => [...recurringKeys.masters(), "list", filters] as const,
  masterDetail: (id: string) => [...recurringKeys.masters(), "detail", id] as const,
  series: () => [...recurringKeys.all, "series"] as const,
  seriesList: (filters?: Record<string, string>) => [...recurringKeys.series(), "list", filters] as const,
  seriesDetail: (id: string) => [...recurringKeys.series(), "detail", id] as const,
  reviewQueue: () => [...recurringKeys.all, "review-queue"] as const,
};
```

**Filter type:**
```typescript
export type RecurringMasterFilters = {
  kind?: string;
  status?: string;
  search?: string;
};
```

**Query hooks (each uses useQuery):**
- `useRecurringDashboard()` — GET /api/recurring/dashboard, staleTime 60_000ms
- `useRecurringMasters(filters)` — GET /api/recurring/masters with query params from filters
- `useRecurringMasterDetail(id)` — GET /api/recurring/masters/{id}, enabled only when id is truthy
- `useRecurringSeries(filters)` — GET /api/recurring/series with query params
- `useRecurringSeriesDetail(id)` — GET /api/recurring/series/{id}, enabled only when id is truthy
- `useReviewQueue()` — GET /api/recurring/review-queue

**Mutation hooks (each uses useMutation with queryClient invalidation):**
- `useCreateMaster()` — POST /api/recurring/masters, invalidates recurringKeys.masters() + recurringKeys.dashboard()
- `useUpdateMaster()` — PATCH /api/recurring/masters/{id}, invalidates recurringKeys.masters() + recurringKeys.masterDetail(id) + recurringKeys.dashboard()
- `useMergeMasters()` — POST /api/recurring/masters/{id}/merge, invalidates recurringKeys.masters() + recurringKeys.dashboard()
- `useChangeMasterStatus()` — POST /api/recurring/masters/{id}/status, invalidates recurringKeys.masters() + recurringKeys.masterDetail(id) + recurringKeys.dashboard()
- `useConfirmSeries()` — POST /api/recurring/series/{id}/confirm, invalidates recurringKeys.series() + recurringKeys.masters() + recurringKeys.reviewQueue() + recurringKeys.dashboard()
- `useIgnoreSeries()` — POST /api/recurring/series/{id}/ignore, invalidates recurringKeys.series() + recurringKeys.reviewQueue() + recurringKeys.dashboard()
- `useResolveReviewItem()` — POST /api/recurring/review-queue/{id}/resolve, invalidates recurringKeys.reviewQueue() + recurringKeys.masters() + recurringKeys.dashboard()
- `useLabelTransaction()` — POST /api/transactions/{id}/label, invalidates ["transactions"] (existing key)

All mutation hooks: show `toast.success()` on success, `toast.error(getErrorMessage(error))` on error. Import toast from sonner, getErrorMessage from @/lib/utils/errors.
