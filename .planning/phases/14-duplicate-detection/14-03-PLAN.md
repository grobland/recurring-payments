<task type="auto">
  <name>Task 2: Create duplicate scan and merge API endpoints</name>
  <files>
    src/app/api/subscriptions/duplicates/route.ts
    src/app/api/subscriptions/merge/route.ts
  </files>
  <action>
    Create POST /api/subscriptions/duplicates (scan endpoint):
    1. Authenticate user
    2. Fetch all active subscriptions (status='active', deletedAt=null, mergedAt=null)
    3. O(n^2) pairwise comparison using calculateSimilarity
    4. Collect pairs with score >= 70%
    5. Sort by score descending
    6. Return array of DuplicatePair objects:
       ```typescript
       interface DuplicatePair {
         sub1: { id, name, amount, currency, frequency, categoryId, createdAt };
         sub2: { id, name, amount, currency, frequency, categoryId, createdAt };
         score: number;
         matches: Record<string, boolean>;
       }
       ```
    7. Add timing log: `console.log(\`Duplicate scan: ${subs.length} subscriptions, ${duplicates.length} pairs found in ${duration}ms\`)`

    Create POST /api/subscriptions/merge (merge endpoint):
    1. Authenticate user
    2. Validate request body:
       ```typescript
       {
         targetId: string;        // Subscription to keep
         sourceId: string;        // Subscription to merge into target
         selectedFields: {        // Which values to use
           name: 'target' | 'source';
           amount: 'target' | 'source';
           frequency: 'target' | 'source';
           category: 'target' | 'source';
           nextRenewalDate: 'target' | 'source';
         };
       }
       ```
    3. Verify both subscriptions exist and belong to user
    4. Update target subscription with selected field values from source if 'source' chosen
    5. Soft delete source: set mergedAt=now(), mergedIntoId=targetId
    6. Return updated target subscription

    Create DELETE /api/subscriptions/merge (undo endpoint):
    1. Authenticate user
    2. Validate request body: { sourceId: string }
    3. Find source subscription by id
    4. Verify it was merged (has mergedAt)
    5. Check 24hr window: if mergedAt > 24 hours ago, return 400 "Undo window expired"
    6. Clear mergedAt and mergedIntoId
    7. Return restored subscription

    Error handling: Use existing patterns (401 unauthorized, 400 validation, 500 server error).
  </action>
  <verify>
    Test endpoints with curl:
    ```bash

    # Merge
    curl -X POST http://localhost:3000/api/subscriptions/merge \
      -H "Content-Type: application/json" \
      -d '{"targetId":"...", "sourceId":"...", "selectedFields":{"name":"target",...}}'
    ```
  </verify>
