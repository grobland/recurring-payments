<feature>
  <name>Multi-Field Weighted Similarity Scoring</name>
  <files>src/lib/utils/similarity.ts, src/lib/utils/similarity.test.ts</files>
  <behavior>
    Expected behavior with test cases:

    1. Exact match returns 100%:
       - Same name, amount, currency, frequency -> 100

    2. Name-only match returns ~50% (name weight):
       - Same name, different everything else -> ~50

    3. Fuzzy name matching with Jaro-Winkler:
       - "Netflix" vs "NETFLIX" -> high name similarity (case insensitive)
       - "Netflix Premium" vs "Netflix" -> partial name match

    4. Amount tolerance (5%):
       - $10.00 vs $10.49 -> amounts match (within 5%)
       - $10.00 vs $11.00 -> amounts don't match (>5% diff)

    5. Currency mismatch:
       - $10 USD vs 10 EUR -> amounts don't match (different currencies)

    6. Missing fields handled gracefully:
       - null categoryId should score neutral (0.5)
       - null statementSource should score neutral (0.5)

    7. Returns matches object showing which fields contributed:
       - { name: true, amount: true, frequency: false, category: false, source: false }
  </behavior>
  <implementation>
    1. Install string-comparison package: `npm install string-comparison`

    2. Create src/lib/utils/similarity.ts:
       - Define SubscriptionRecord interface matching schema fields
       - Define SimilarityWeights interface with defaults
       - Define SimilarityResult interface: { score: number, matches: Record<string, boolean> }
       - Implement calculateSimilarity function:
         a. Name: Jaro-Winkler on .toLowerCase() strings
         b. Amount: Exact match with 5% tolerance (only compare if same currency)
         c. Frequency: Exact match (monthly/yearly)
         d. Category: Exact match (null-safe)
         e. Source: Jaro-Winkler if both exist, 0.5 neutral if either missing
       - Weighted sum, convert to 0-100 percentage
       - Return score and matches object

    3. Create src/lib/utils/similarity.test.ts:
       - Test exact match scenario
       - Test partial match (name only)
       - Test case insensitivity
       - Test amount tolerance boundary (4.9% vs 5.1%)
       - Test currency mismatch
       - Test null handling for optional fields
       - Test matches object accuracy
  </implementation>
</feature>
