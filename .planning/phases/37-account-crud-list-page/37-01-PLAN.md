**Create `src/app/api/accounts/route.ts`** with GET and POST handlers following the tags route pattern exactly:

**GET /api/accounts:**
- Auth check (`session?.user?.id`)
- `db.query.financialAccounts.findMany` where userId matches session, ordered by `[asc(financialAccounts.accountType), asc(financialAccounts.name)]`
- Return `{ accounts }` (array of FinancialAccount)

**POST /api/accounts:**
- Auth check + `isUserActive` check (403 if expired)
- Parse body with `createAccountSchema.safeParse(body)`; return 400 on failure
- **Source link conflict check (ACCT-07):** If `linkedSourceType` is provided, check if another account for this user already has the same `linkedSourceType`. If yes, return 409: `Source "${linkedSourceType}" is already linked to another account`.
- **interestRate conversion:** If `accountType === "loan"` and `interestRate` is provided, divide by 100 before storing: `interestRate: String(Number(result.data.interestRate) / 100)`. The DB expects a decimal string.
- `db.insert(financialAccounts).values({ userId, ...data, updatedAt: new Date() }).returning()`
- **Bulk link existing statements (ACCT-07):** If `linkedSourceType` was provided, bulk-update existing unlinked statements: `db.update(statements).set({ accountId: account.id }).where(and(eq(statements.userId, ...), eq(statements.sourceType, linkedSourceType), isNull(statements.accountId)))`. Import `isNull` from `drizzle-orm`.
- Return `{ account }` with status 201

**Create `src/app/api/accounts/[id]/route.ts`** with GET, PATCH, DELETE:

```typescript
interface RouteParams { params: Promise<{ id: string }>; }
```

**GET /api/accounts/[id]:**
- Auth check + `await params` to get `id`
- `db.query.financialAccounts.findFirst` where id AND userId match
- 404 if not found; return `{ account }`

**PATCH /api/accounts/[id]:**
- Auth check + `isUserActive` + `await params`
- Parse body with `updateAccountSchema.safeParse(body)`; 400 on failure
- Find existing account (ownership check, 404 if missing)
- **Strip accountType from update data** — type is locked during edit per CONTEXT. Delete `result.data.accountType` before spreading into the update.
- **Source link conflict check:** If `linkedSourceType` is being changed and is not null, check if another account claims it (exclude self with `ne(financialAccounts.id, id)`). 409 if conflict.
- **interestRate conversion:** Same as POST — if interestRate provided, divide by 100.
- `db.update(financialAccounts).set({ ...data, updatedAt: new Date() }).where(eq(financialAccounts.id, id)).returning()`
- **Bulk-update statements on source link change:** If `linkedSourceType` changed from the existing value:
  - If old linkedSourceType existed: set accountId to NULL on statements with old sourceType for this user (unlink old)
  - If new linkedSourceType exists: set accountId to account.id on unlinked statements with new sourceType for this user
- Return `{ account: updated }`

**DELETE /api/accounts/[id]:**
- Auth check + `isUserActive` + `await params`
- Find existing (ownership check, 404 if missing)
- `db.delete(financialAccounts).where(and(eq(financialAccounts.id, id), eq(financialAccounts.userId, session.user.id)))`
- statements.account_id becomes NULL via DB `onDelete: "set null"` — no manual statement update needed
- Return `{ message: "Account deleted" }`

All handlers wrapped in try/catch with `console.error` + 500 fallback, matching tags route pattern.
