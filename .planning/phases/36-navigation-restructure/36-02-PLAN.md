**3. Add isNavItemActive helper function**

Add this helper function just before the `AppSidebar` component declaration:

```typescript
function isNavItemActive(pathname: string, href: string): boolean {
  // Items with child routes use prefix match; others use exact match
  const prefixMatchItems = ["/payments/subscriptions", "/settings"];
  if (prefixMatchItems.includes(href)) {
    return pathname === href || pathname.startsWith(`${href}/`);
  }
  return pathname === href;
}
```

This prevents `/vault` from activating when at `/vault/load` (they are siblings, not parent-child). Only `/payments/subscriptions` and `/settings` genuinely have child routes that should keep the parent item active.
