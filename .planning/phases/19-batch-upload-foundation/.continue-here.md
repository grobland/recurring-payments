<decisions_made>

- Used queueRef pattern: React setState callback is async, can't read state synchronously in while loop. Solution: keep ref in sync with state, use ref.current for reads.
- Sequential processing: One file at a time prevents memory exhaustion with large PDFs (50-100MB)
- Server-side PDF parsing: File sent to /api/batch/process via FormData, server uses pdf2json
- User-scoped uniqueness: pdfHash is unique per user, not globally (allows different users to upload same statement)
- Re-import disabled: Skip button works, Re-import button disabled with "coming soon" (requires blob storage)
</decisions_made>
