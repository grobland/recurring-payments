<task type="auto">
  <name>Task 3: Complete deployment and add environment variables</name>
  <files>none</files>
  <action>
If not already deployed, run deployment:

```bash
npx vercel --yes
```

Capture the preview URL from output (format: https://project-name-xxx.vercel.app).

Next, add environment variables to the Vercel project. The Vercel CLI can add env vars:

```bash
# Add each environment variable for Preview environment
# These need to match what's in .env.local

echo "$OPENAI_API_KEY" | npx vercel env add OPENAI_API_KEY preview
echo "$STRIPE_SECRET_KEY" | npx vercel env add STRIPE_SECRET_KEY preview
echo "$STRIPE_PUBLISHABLE_KEY" | npx vercel env add STRIPE_PUBLISHABLE_KEY preview
echo "$RESEND_API_KEY" | npx vercel env add RESEND_API_KEY preview
```

If CLI env commands fail, provide instructions for user to add via dashboard:
1. Go to Vercel Dashboard -> Project -> Settings -> Environment Variables
2. Add each variable for "Preview" environment

After adding env vars, trigger a new deployment to pick them up:

```bash
npx vercel --yes --prod=false
```

Capture the final preview URL.
  </action>
  <verify>Deployment completes with preview URL, environment variables added</verify>
  <done>Preview URL obtained and environment variables configured in Vercel</done>
