  <action>
Fix the stale closure bug in handleBulkTag callback.

Problem (lines 109-122):
```typescript
const handleBulkTag = useCallback(
  (tagId: string) => {
    const transactionIds = Array.from(selectedIds);  // STALE - captured at callback creation
    bulkTagMutation.mutate(...)
  },
  [selectedIds, bulkTagMutation]  // Re-creates callback, but timing issue persists
);
```

Solution - use ref pattern:

1. Add useRef import (should already exist, verify)

2. After the selectedIds state declaration, add ref:
```typescript
const [selectedIds, setSelectedIds] = useState<Set<string>>(new Set());
const selectedIdsRef = useRef(selectedIds);
```

3. Add useEffect to sync ref with state:
```typescript
useEffect(() => {
  selectedIdsRef.current = selectedIds;
}, [selectedIds]);
```

4. Update handleBulkTag to read from ref:
```typescript
const handleBulkTag = useCallback(
  (tagId: string) => {
    const transactionIds = Array.from(selectedIdsRef.current);  // READ FROM REF
    bulkTagMutation.mutate(
      { transactionIds, tagId, action: "add" },
      {
        onSuccess: () => {
          setSelectedIds(new Set());
        },
      }
    );
  },
  [bulkTagMutation]  // REMOVE selectedIds from dependencies
);
```

Key changes:
- selectedIdsRef.current gives sync access to latest state
- Remove selectedIds from useCallback dependencies
- Keep bulkTagMutation in dependencies (it's stable from useMutation)
- Ref update happens synchronously before any callback invocation

This pattern is established in the codebase (see STATE.md: "queueRef pattern: Use ref for sync state access in async loops").
