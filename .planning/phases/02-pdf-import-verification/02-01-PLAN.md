  <files>src/app/api/import/route.ts</files>
  <action>
Improve error handling in the import API route to provide user-friendly error messages for common OpenAI API failures:

1. Import OpenAI at the top: `import OpenAI from "openai";`
2. In the catch block, check for specific OpenAI error types:
   - Rate limit (429): "Service temporarily busy. Please try again in a moment."
   - Timeout (408 or timeout in message): "Processing took too long. Please try with a smaller file."
   - Other API errors: Use the existing generic message

The error handling pattern from RESEARCH.md:
```typescript
if (error instanceof OpenAI.APIError) {
  if (error.status === 429) {
    return NextResponse.json(
      { error: "Service temporarily busy. Please try again in a moment." },
      { status: 429 }
    );
  }
  if (error.status === 408 || error.message?.includes('timeout')) {
    return NextResponse.json(
      { error: "Processing took too long. Please try with a smaller file." },
      { status: 408 }
    );
  }
}
```

Keep the existing generic 500 error as fallback for unexpected errors.
