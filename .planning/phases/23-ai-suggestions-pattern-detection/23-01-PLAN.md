  <action>
Create POST /api/patterns/bulk endpoint that handles bulk accept and dismiss operations.

**Request schema:**
```typescript
{
  patternIds: string[];
  action: "accept" | "dismiss";
}
```

**Implementation requirements:**
1. Validate session (401 if unauthorized)
2. Validate request body with Zod schema
3. Use database transaction with SELECT FOR UPDATE to prevent race conditions
4. For "accept" action:
   - Fetch all patterns with given IDs that are not already accepted/dismissed
   - Verify all patterns belong to current user
   - For each pattern, create subscription using same logic as single accept:
     - Use guessCategory for categoryId
     - Calculate nextRenewalDate from most recent charge + frequency
     - Set normalizedMonthlyAmount
   - Mark all patterns as acceptedAt with createdSubscriptionId
5. For "dismiss" action:
   - Update all patterns to set dismissedAt = now()
6. Return { success: true, acceptedCount/dismissedCount, subscriptionIds? }

**Key patterns to follow:**
- Use inArray() for bulk pattern fetch
- Wrap all mutations in db.transaction() for atomicity
- Use sql`FOR UPDATE` to lock rows during transaction
- Return early if patternIds array is empty

**Error handling:**
- 400 for invalid body or empty array
- 404 if no valid patterns found (all already processed or wrong user)
- 500 for transaction failure
