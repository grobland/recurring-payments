<task type="auto">
  <name>Task 1: Create error utilities module</name>
  <files>src/lib/utils/errors.ts</files>
  <action>
Create a new error utilities module with two exported functions:

1. `getErrorMessage(error: unknown): string`
   - Transform technical errors to user-friendly messages
   - Handle specific patterns:
     - "Unauthorized" or 401 -> "Your session has expired. Please sign in again."
     - "Invalid category" -> "The selected category is not valid. Please choose another."
     - "trial has expired" -> "Your trial has expired. Please upgrade to continue."
     - "Failed to fetch" or "Network" or TypeError -> "Unable to connect. Please check your internet connection and try again."
     - "too large" -> "File too large. Please use a smaller file."
     - "Invalid PDF" -> "Invalid PDF format. Please upload a valid PDF file."
     - "No transactions" -> "No transactions found. Try a different file or add subscriptions manually."
     - "OpenAI" or "AI service" -> "AI service temporarily unavailable. Please try again in a moment."
     - 503 status -> "Service temporarily unavailable. Please try again in a moment."
   - Generic fallback: "Something went wrong. Please try again."

2. `isRetryableError(error: unknown): boolean`
   - Return true for transient failures that should trigger automatic retry:
     - Network errors (message includes "Failed to fetch" or "Network" or TypeError)
     - 503 Service Unavailable
     - 408 Request Timeout
   - Return false for all other errors (validation, auth, etc.)

Include proper TypeScript types. The error parameter can be Error, Response, or unknown.
  </action>
  <verify>
Run `npx tsc --noEmit` to verify no TypeScript errors.
Create a quick test by importing the functions and logging sample outputs.
  </verify>
  <done>
getErrorMessage transforms various error types to friendly messages.
isRetryableError correctly identifies network errors and 503 responses.
  </done>
</task>
