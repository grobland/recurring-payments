  <action>
Create a GitHub-style heat map grid component. This is the core visual component of the coverage feature.

**Component interface:**
```typescript
interface CoverageCellClickInfo {
  sourceType: string;
  month: string;           // "2026-02"
  state: "pdf" | "data" | "missing";
  statementId: string | null;
}

interface CoverageGridProps {
  sources: CoverageSource[];
  months: string[];          // 12 month labels from API
  onCellClick: (info: CoverageCellClickInfo) => void;
}
```

**Layout using CSS Grid** (proven pattern from `timeline-grid.tsx`):
- Grid template: `auto repeat(12, minmax(0, 1fr))` -- first column for source names, then 12 month columns
- Source name column: `text-sm font-medium`, truncated with `truncate max-w-[120px]` for long names
- Month header row: `text-xs text-center text-muted-foreground`, show abbreviated month name using `format(parseISO(month + "-01"), "MMM")` from date-fns

**Cell rendering:**
- Each cell is a `button` element (all three states are interactive per user decision)
- Minimum size: `min-w-7 min-h-7` (28px touch targets per research)
- Rounded corners: `rounded-sm` for compact GitHub-style look
- Cell color classes using Tailwind:
  - `pdf`: `bg-green-500/80 hover:bg-green-500`
  - `data`: `bg-yellow-400/80 hover:bg-yellow-400`
  - `missing`: `bg-muted/40 hover:bg-muted/60`
- All cells: `cursor-pointer transition-colors`
- Use `cn()` from `@/lib/utils` for conditional classes

**Tooltip on hover** using shadcn Tooltip:
- Wrap the entire grid in a single `TooltipProvider` (NOT one per cell -- that's expensive)
- Each cell wrapped in `Tooltip` + `TooltipTrigger` (asChild on the button) + `TooltipContent`
- Tooltip content shows:
  - For "pdf" cells: "PDF stored" + statement date (formatted as "MMM d, yyyy") + "{N} transactions"
  - For "data" cells: "Data only - no PDF" + statement date + "{N} transactions"
  - For "missing" cells: "No data - click to upload"
- Use `delayDuration={300}` on TooltipProvider for snappy but non-jittery feel

**Legend bar** (rendered below the grid):
- Horizontal flex row with three items: colored circle + label
- `<span className="inline-block w-3 h-3 rounded-sm bg-green-500/80" />` "PDF stored"
- `<span className="inline-block w-3 h-3 rounded-sm bg-yellow-400/80" />` "Data only"
- `<span className="inline-block w-3 h-3 rounded-sm bg-muted/40 border" />` "Missing"
- Wrap in `flex items-center gap-4 text-xs text-muted-foreground mt-3`

**Accessibility:**
- Each cell button has `aria-label` describing the state: "{source} - {month formatted as MMM yyyy}: {state description}"
- Grid has `role="grid"` with source rows having `role="row"`

Import types `CoverageSource`, `CoverageCell` from `@/lib/hooks/use-vault-coverage`.
Import `format`, `parseISO` from `date-fns`.
Import `Tooltip`, `TooltipContent`, `TooltipProvider`, `TooltipTrigger` from `@/components/ui/tooltip`.

Create the upload wizard modal following the Dialog pattern from `pdf-viewer-modal.tsx` and dropzone pattern from `batch-uploader.tsx`.

**Component interface:**
```typescript
interface HistoricalUploadModalProps {
  open: boolean;
  onClose: () => void;
  sourceType: string;
  targetMonth: string;        // "2026-02"
  mode: "missing" | "attach"; // missing = gray cell (new upload), attach = yellow cell (add PDF)
  statementId?: string;       // present when mode === "attach"
}
```

**Modal structure:**
- shadcn `Dialog` with `DialogContent className="sm:max-w-md"`
- `DialogHeader` with `DialogTitle`:
  - mode "attach": "Attach PDF - {sourceType}"
  - mode "missing": "Upload Statement - {sourceType}"
- Below title, show the target month formatted as "MMMM yyyy" (e.g., "February 2026") using `format(parseISO(targetMonth + "-01"), "MMMM yyyy")`

**Mode-specific messaging:**
- If mode === "attach": Show a muted info message above the dropzone: "You have data for this month but no PDF. Drop a file below to attach the original statement."
- If mode === "missing": Show: "Upload a statement PDF for this source and month."

**Dropzone area:**
- Use `react-dropzone` with `useDropzone` hook
- Accept: `{ "application/pdf": [".pdf"] }`
- maxFiles: 1
- maxSize: `50 * 1024 * 1024` (50MB, matching batch upload limit)
- Dropzone container: bordered dashed area, `min-h-[120px]`, centered content
- States:
  - Default: Upload icon + "Drop PDF here or click to browse"
  - Drag active: `border-primary bg-primary/5` + "Drop file here"
  - File selected: Show filename + file size + "x" button to clear
- Use `Upload` and `X` icons from lucide-react

**Submit flow:**
- "Upload" button below dropzone, disabled when no file or isUploading
- Show `Loader2` spinner with `animate-spin` when uploading
- State management: `file`, `isUploading`, `error` via useState

**Mode "attach" submit:**
```
const formData = new FormData();
formData.append("file", file);
formData.append("statementId", statementId!);
const res = await fetch("/api/vault/attach-pdf", { method: "POST", body: formData });
```

**Mode "missing" submit:**
```
const formData = new FormData();
formData.append("file", file);
formData.append("sourceType", sourceType);
const { hashFile } = await import("@/lib/utils/file-hash");
const hash = await hashFile(file);
formData.append("hash", hash);
const res = await fetch("/api/batch/upload", { method: "POST", body: formData });
```
Note: For gray cell uploads, the batch upload endpoint creates the statement record + stores the PDF. Processing (transaction extraction) is NOT triggered from the wizard -- the user can do that separately from the batch import page if desired. This keeps the wizard simple and focused on PDF storage.

**On success:**
1. `queryClient.invalidateQueries({ queryKey: ["vault", "coverage"] })` -- grid updates
2. Also invalidate `["vault", "timeline"]` and `["sources"]` so other vault tabs refresh
3. `toast.success("PDF stored successfully")` using sonner
4. Call `onClose()` to dismiss modal
5. Reset file and error state

**On error:**
- Set error message from response JSON or generic "Upload failed"
- Show error in red text below the dropzone: `<p className="text-sm text-destructive">{error}</p>`
- Include a "Retry" option (button that clears error, keeps file selected)

**Reset on close:** When modal closes (onOpenChange false), reset file, error, and uploading states.

Import `useQueryClient` from `@tanstack/react-query`, `toast` from `sonner`, Dialog components from shadcn, `useDropzone` from `react-dropzone`, `Button` from shadcn, icons from lucide-react, `cn` from `@/lib/utils`, `format`/`parseISO` from `date-fns`.
