Create a raw SQL migration file that:

1. Creates the materialized view `user_analytics_mv` with these columns:
   - user_id (UUID)
   - category_id (UUID, nullable for uncategorized)
   - category_name (VARCHAR)
   - category_color (VARCHAR)
   - category_icon (VARCHAR)
   - month (TIMESTAMP - from DATE_TRUNC('month', next_renewal_date AT TIME ZONE 'UTC'))
   - currency (VARCHAR)
   - subscription_count (INTEGER)
   - total_amount (NUMERIC)
   - normalized_monthly_amount (NUMERIC)

2. SELECT from subscriptions LEFT JOIN categories, filtering:
   - WHERE deleted_at IS NULL AND status = 'active'

3. GROUP BY user_id, category columns, month, currency

4. Create unique index for CONCURRENTLY refresh:
   - Use COALESCE for nullable category_id: `COALESCE(category_id, '00000000-0000-0000-0000-000000000000'::uuid)`
   - Index on (user_id, coalesced_category_id, month, currency)

5. Create additional indexes for query performance:
   - user_analytics_mv_user_id_idx on user_id
   - user_analytics_mv_month_idx on month

6. Include initial REFRESH to populate data

Follow PostgreSQL best practices from research. Use AT TIME ZONE 'UTC' for date truncation to avoid timezone issues.
