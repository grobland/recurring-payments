<task type="auto">
  <name>Task 2: Create analytics API endpoint</name>
  <files>src/app/api/analytics/route.ts</files>
  <action>
Create GET /api/analytics endpoint following existing API patterns:

1. Import auth from @/lib/auth, db from @/lib/db, sql from drizzle-orm
2. Import getExchangeRates, convertCurrency from @/lib/fx/rates
3. Import Zod for param validation
4. Import users from @/lib/db/schema

5. Create Zod schema for query params:
   - period: enum ['month', 'year', 'quarter'], default 'month'
   - year: coerce to number, min 2020, max 2100, default current year
   - month: optional coerce to number, 1-12
   - quarter: optional coerce to number, 1-4

6. GET handler implementation:
   a. Authenticate user (return 401 if not authenticated)
   b. Parse and validate query params
   c. Build date filter based on period:
      - month: first day of month to first day of next month
      - year: Jan 1 to Jan 1 next year
      - quarter: first month of quarter to first month of next quarter
   d. Query user_analytics_mv using raw SQL (Drizzle can query views):
      ```sql
      SELECT * FROM user_analytics_mv
      WHERE user_id = $1 AND month >= $2 AND month < $3
      ```
   e. Get user's displayCurrency from users table
   f. Get exchange rates from getExchangeRates()
   g. Process results:
      - Group by category
      - Convert each amount to display currency
      - Track original currency breakdown
      - Calculate totals and percentages
   h. Return AnalyticsResponse JSON

7. Error handling:
   - 401 for unauthenticated
   - 400 for invalid params
   - 500 for server errors with logged details

8. Add response headers for caching:
   - Cache-Control: private, max-age=300 (5 minutes)
  </action>

<task type="auto">
  <name>Task 3: Create useAnalytics React hook</name>
  <files>src/lib/hooks/use-analytics.ts</files>
  <action>
Create TanStack Query hook following use-subscriptions.ts patterns:

1. Import useQuery, useQueryClient from @tanstack/react-query
2. Import AnalyticsResponse, AnalyticsPeriod from @/types/analytics
3. Import isRetryableError from @/lib/utils/errors

4. Create query key factory:
   ```typescript
   export const analyticsKeys = {
     all: ['analytics'] as const,
     period: (period: AnalyticsPeriod, year: number, month?: number, quarter?: number) =>
       [...analyticsKeys.all, { period, year, month, quarter }] as const,
   };
   ```

5. Create fetchAnalytics async function:
   - Build query string from params
   - Fetch from /api/analytics
   - Handle non-ok response by throwing error
   - Return typed AnalyticsResponse

6. Create useAnalytics hook:
   ```typescript
   export function useAnalytics(
     period: AnalyticsPeriod,
     year: number,
     month?: number,
     quarter?: number
   )
   ```
   - Use useQuery with:
     - queryKey: analyticsKeys.period(...)
     - queryFn: fetchAnalytics(...)
     - staleTime: 5 * 60 * 1000 (5 minutes)
     - refetchOnWindowFocus: false (data refreshes server-side)
     - retry: use isRetryableError for retry decision, max 2 retries
   - Return typed query result

7. Create useInvalidateAnalytics hook:
   - Get queryClient
   - Return function that calls queryClient.invalidateQueries({ queryKey: analyticsKeys.all })

8. Export analyticsKeys, useAnalytics, useInvalidateAnalytics
