   - Query user's categories (including defaults)
   - Use simple keyword matching:
     - "netflix", "hulu", "disney", "spotify", "youtube" -> "Entertainment" category
     - "aws", "azure", "google cloud", "digitalocean" -> "Software" category

   - Return categoryId or null if no match

2. Create src/app/api/transactions/[id]/convert/route.ts:

   POST handler (convert transaction to subscription):
   - Authenticate user
   - Fetch transaction with userId check (security)
   - If already converted (convertedToSubscriptionId not null), return 400 "Already converted"
   - Guess category from merchant name
   - Calculate next renewal date: add 1 month to transaction.transactionDate
   - Create subscription with:
     - name: transaction.merchantName
     - amount: transaction.amount
     - currency: transaction.currency
     - frequency: "monthly" (default)
     - nextRenewalDate: calculated
     - categoryId: guessed category or null
     - status: "active"
     - reminderEnabled: true
   - Update transaction:
     - convertedToSubscriptionId: new subscription ID
     - tagStatus: "converted"
   - Return { success: true, subscription, transaction }

   DELETE handler (undo conversion):
   - Authenticate user
   - Fetch transaction with userId check
   - If not converted (convertedToSubscriptionId is null), return 400 "Not converted"
   - Store subscriptionId before clearing
   - Update transaction:
     - convertedToSubscriptionId: null
     - tagStatus: "unreviewed"
   - Optionally delete the subscription (or leave for user to manage)
   - For simplicity: delete the subscription (it was just created)
   - Return { success: true, transaction }

Use transactions with Drizzle for atomicity.
