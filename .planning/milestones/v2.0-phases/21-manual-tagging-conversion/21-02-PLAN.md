   - PATCH handler for toggling a tag on/off:
     - Authenticate user
     - Validate body: { tagId: string, action: "add" | "remove" }
     - Verify transaction belongs to user (query with userId check)
     - Verify tag belongs to user
     - If action is "add": insert into transactionTags (ignore if exists - upsert pattern)
     - If action is "remove": delete from transactionTags where transactionId and tagId match
     - Return { success: true, transaction: TransactionWithTags }
     - Handle errors: 401, 404 (transaction not found), 400 (invalid tag)

     - mutationFn: PATCH to /api/transactions/[id]/tags with { tagId, action }
     - onSuccess: invalidate transactionKeys.lists() to refresh transaction data
     - Export for use in TagCombobox

Follow existing mutation patterns from use-categories.ts.
