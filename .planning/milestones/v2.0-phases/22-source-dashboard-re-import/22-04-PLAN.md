Create useIncompleteBatch hook that detects and provides access to incomplete batch state from localStorage.

The hook should:
1. Read from localStorage key "batch-upload-queue" (same as use-batch-upload.ts)
2. Parse the stored data to detect incomplete state
3. Return null if no incomplete batch or data expired (24 hour expiry)
4. Return IncompleteBatch object with:
   - sourceType: string
   - timestamp: number
   - processed: number (count of complete/error files)
   - total: number (total queue length)
   - lastError: string | undefined (error message from failed file)
   - completedFiles: { filename: string; status: string }[] (successfully processed)
   - pendingFiles: { filename: string }[] (not yet processed)

Provide discard function that:
1. Removes item from localStorage
2. Clears the batch state

Handle hydration properly:
- Initialize state as null
- Update in useEffect to avoid SSR/client mismatch
- Use useState with lazy initialization

```typescript
export interface IncompleteBatch {
  sourceType: string;
  timestamp: number;
  processed: number;
  total: number;
  lastError?: string;
  completedFiles: { filename: string; status: string }[];
  pendingFiles: { filename: string }[];
}

export function useIncompleteBatch(): {
  batch: IncompleteBatch | null;
  discard: () => void;
}
```
