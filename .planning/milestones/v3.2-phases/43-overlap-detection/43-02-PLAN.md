Create `src/lib/hooks/use-overlap-dismissals.ts` as a React hook (no "use client" directive needed — works in any client component context).

**Group signature:** A stable string that uniquely identifies the current members of an overlap group. Compute as: `sorted(subscriptionIds).join(",")`. If the signature stored in localStorage differs from the current signature, the group is re-surfaced (not dismissed).

**localStorage schema:**
```typescript
// Key: "overlap_dismissals"
// Value: JSON-stringified Record<categoryId, groupSignature>
// Example: { "uuid-entertainment": "id1,id2", "uuid-streaming": "id3,id4,id5" }
```

**Hook signature:**
```typescript
export function useOverlapDismissals(overlapGroups: Map<string, string[]>): {
  isDismissed: (categoryId: string) => boolean;
  dismiss: (categoryId: string) => void;
}
```

**isDismissed(categoryId) logic:**
1. Read localStorage "overlap_dismissals" (parse JSON, default to `{}` on error)
2. If categoryId not in stored record → return false (not dismissed)
3. Get current group's subscriptionIds from `overlapGroups.get(categoryId)`
4. If group no longer exists (dissolved) → delete that key from localStorage, return false
5. Compute current signature: `[...ids].sort().join(",")`
6. If stored signature !== current signature → return false (re-surface: members changed)
7. If signatures match → return true (still dismissed, same members)

**dismiss(categoryId) logic:**
1. Get current subscriptionIds from `overlapGroups.get(categoryId)` — return early if not found
2. Compute signature: `[...ids].sort().join(",")`
3. Read current localStorage value (parse, default to `{}`)
4. Add/update entry: `{ ...existing, [categoryId]: signature }`
5. Write back to localStorage
6. Trigger re-render: use `useState` for a counter that increments on dismiss, so `isDismissed` recomputes

**Cleanup:** When `overlapGroups` changes (useEffect or on each render), scan stored keys and remove any categoryId whose group no longer exists in `overlapGroups` (prevents stale data buildup).

Use `useState` (not `useRef`) for the dismissals state to ensure re-renders on dismiss. Read from localStorage lazily (only in `isDismissed` calls, or initialize state from localStorage on mount).

Prefer simplicity: initialize `dismissals` state from localStorage on mount with `useState(() => { ... })` lazy initializer. On `dismiss`, update both state and localStorage. On `isDismissed`, use the React state (not re-reading localStorage) for synchronous consistency.

{(() => {
  // Find which overlap group this subscription belongs to
  const overlappingGroupId = subscription.categoryId
    ? (overlapGroups.has(subscription.categoryId) ? subscription.categoryId : null)
    : null;

  if (!overlappingGroupId) return null;
  if (isDismissed(overlappingGroupId)) return null;

  // Names of the OTHER subscriptions in this group (not this one)
  const groupIds = overlapGroups.get(overlappingGroupId) ?? [];
  const otherNames = subscriptions
    .filter(s => groupIds.includes(s.id) && s.id !== subscription.id)
    .map(s => s.name);

  return (
    <OverlapBadge
      key={overlappingGroupId}
      categoryId={overlappingGroupId}
      otherNames={otherNames}
      onDismiss={dismiss}
    />
  );
})()}
