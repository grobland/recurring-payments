FIX - Check BOTH scopes in the query itself:
```typescript
// Check if slug conflicts with default categories OR user's own custom categories
const existing = await db.query.categories.findFirst({
  where: and(
    eq(categories.slug, slug),
    or(
      isNull(categories.userId),              // Would conflict with default category
      eq(categories.userId, session.user.id)  // Would conflict with own custom category
    )
  )
});

if (existing) {
  return NextResponse.json(
    { error: "A category with this name already exists" },
    { status: 409 }
  );
}
```

Add `and` to the imports from drizzle-orm (should already have `or` and `isNull`).

Why this works: The original code finds ANY category with matching slug, then checks ownership afterward. This is wrong because if another user created a category with that slug first, it wouldn't find it as a conflict. The fix queries for conflicts in a single WHERE clause that properly scopes to default + current user's categories.
