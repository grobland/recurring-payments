# Milestone v1.1: Import Improvements

**Status:** SHIPPED 2026-02-02
**Phases:** 5-8
**Total Plans:** 11

## Overview

Improve PDF import accuracy and user control, fix data quality issues discovered during testing.

**Target features:**
- Smart import: Show all statement items with confidence scores, user selects which to import
- Statement sources: Track bank/credit card name, reuse for future imports
- Renewal date fix: Calculate from actual transaction dates on statements
- Category management: Fix duplicates, add full CRUD for categories

## Phases

### Phase 5: Category Management

**Goal**: Users can manage categories without duplicates and with full CRUD operations
**Depends on**: Nothing (first phase of v1.1, independent of other phases)
**Plans**: 3 plans

Plans:
- [x] 05-01-PLAN.md — Fix duplicate validation bug in POST /api/categories
- [x] 05-02-PLAN.md — Create CategoryCombobox component and integrate into subscription form
- [x] 05-03-PLAN.md — Build category CRUD UI (form, delete dialog, manager)

**Details:**
- API validates duplicates using and/or clause for user-scoped and default categories
- CategoryCombobox uses Command pattern for searchable dropdown
- Delete operation properly nullifies categoryId on affected subscriptions
- TanStack Query cache invalidation provides instant UI updates

### Phase 6: Statement Source Tracking

**Goal**: Users can track which bank or credit card each statement came from
**Depends on**: Phase 5 (independent work, but sequenced for organized delivery)
**Plans**: 3 plans

Plans:
- [x] 06-01-PLAN.md — Schema and API foundation (statementSource column, sources endpoint)
- [x] 06-02-PLAN.md — Import UI integration (AccountCombobox, useImportSources hook)
- [x] 06-03-PLAN.md — Display source in subscription detail page

**Details:**
- AccountCombobox with autocomplete from /api/import/sources
- statementSource column in import_audits schema
- Subscription detail page displays source via importAudit relation
- Case-insensitive deduplication of previous sources

### Phase 7: Smart Import UX

**Goal**: Users see all detected statement items and can choose which ones to import
**Depends on**: Phase 6 (builds on statement source tracking foundation)
**Plans**: 3 plans

Plans:
- [x] 07-01-PLAN.md — Schema and API for raw extraction data persistence
- [x] 07-02-PLAN.md — Badge component success/warning variants
- [x] 07-03-PLAN.md — Import page UI with confidence badges, bulk selection, inline editing

**Details:**
- AI prompt explicitly requests ALL items regardless of confidence
- Threshold adjusted to 80/50 (vs spec 70/40) for better accuracy
- rawExtractionData persisted as JSONB for audit trail
- Bulk selection controls (selectAll, selectNone, selectHighConfidence)

### Phase 8: Renewal Date Intelligence

**Goal**: Renewal dates are calculated from actual transaction dates on statements
**Depends on**: Phase 7 (builds on AI extraction enhancements)
**Plans**: 2 plans

Plans:
- [x] 08-01-PLAN.md — Extend AI extraction for transaction dates and add renewal calculation utility
- [x] 08-02-PLAN.md — Import UI with date display, inline editing, and visual diff

**Details:**
- AI extracts transactionDate in ISO 8601 format
- calculateRenewalFromTransaction advances date by billing cycle
- EditableDateField with visual diff and restore capability
- Auto-recalculation when transaction date changes

## Milestone Summary

**Key Decisions:**
- AI prompt returns ALL items with confidence scores (changed from "be conservative")
- Confidence thresholds: 80+ (high), 50-79 (medium), 0-49 (low)
- Transaction date is source of truth for renewal calculation
- Click-to-edit pattern for inline date fields
- Command palette pattern for searchable selectors

**Issues Resolved:**
- PDF import missing low/medium confidence items — fixed AI prompt
- Category dropdown duplicate entries — fixed validation logic
- Renewal dates incorrect — now calculated from transaction dates
- No statement source tracking — added statementSource field and autocomplete

**Issues Deferred:**
- Multi-statement pattern detection — requires historical data
- AI confidence calibration dashboard — needs usage data first

**Technical Debt Incurred:**
- None — all implementations complete with proper error handling and type safety
