# Milestone v2.0: Statement Hub

**Status:** SHIPPED 2026-02-10
**Phases:** 19-23
**Total Plans:** 21 (19 + 2 gap closures)

## Overview

Transform import from single-file extraction to comprehensive statement management system with batch uploads, full data retention, and manual enrichment capabilities.

## Phases

### Phase 19: Batch Upload Foundation

**Goal**: Users can upload multiple PDFs at once and system stores all statement line items with robust deduplication
**Depends on**: Phase 18
**Requirements**: BATCH-01, BATCH-02, BATCH-03, BATCH-04, BATCH-05, DATA-01, DATA-02, DATA-03, DATA-04
**Plans**: 5 plans

Plans:
- [x] 19-01: Schema + file hashing utility (statements, transactions tables)
- [x] 19-02: Batch API endpoints (check-hash, upload, process)
- [x] 19-03: useBatchUpload hook (queue management, sequential processing)
- [x] 19-04: UI components (BatchUploader, FileQueue, FileItem)
- [x] 19-05: Integration + verification (batch import page, navigation)

**Details:**
- Chunked file hashing (1MB chunks) for memory efficiency
- User-scoped PDF hash uniqueness
- Transaction fingerprinting (merchant+amount+date) for dedup
- Sequential processing to prevent memory exhaustion
- LocalStorage queue persistence for resume capability

### Phase 20: Statement Browser & Filtering

**Goal**: Users can browse and filter all stored transactions with fast, responsive UI
**Depends on**: Phase 19
**Requirements**: BRWS-01, BRWS-02, BRWS-03, BRWS-04, BRWS-05, BRWS-06
**Plans**: 2 plans

Plans:
- [x] 20-01: API endpoint + useTransactions hook with keyset pagination
- [x] 20-02: Virtualized table/cards + filters + /transactions page

**Details:**
- Keyset pagination (date, id) cursor for O(1) performance at any depth
- TanStack Virtual for virtualized scrolling (10k+ items at 60fps)
- Responsive table (desktop) and card (mobile) layouts
- Debounced search and filter controls

### Phase 21: Manual Tagging & Conversion

**Goal**: Users can manually enrich statement data by tagging items and converting them to subscriptions
**Depends on**: Phase 20
**Requirements**: ENRCH-01, ENRCH-02, ENRCH-03, ENRCH-04
**Plans**: 4 plans + 2 gap closures

Plans:
- [x] 21-01: Tags schema, CRUD API, and hooks
- [x] 21-02: Tag UI components and inline tagging
- [x] 21-03: Conversion API and one-click flow
- [x] 21-04: Bulk selection and action bar
- [x] 21-05: Tag Management UI (gap closure)
- [x] 21-06: Fix stale closure in bulk tag (gap closure)

**Details:**
- Many-to-many junction table (transactionTags)
- Inline TagCombobox in table rows and cards
- One-click conversion with 8-second undo toast
- Bulk operations with floating action bar
- selectedIdsRef pattern for sync access in async callbacks

### Phase 22: Source Dashboard & Re-import

**Goal**: Users can see overview of statement coverage and re-import items they initially skipped
**Depends on**: Phase 21
**Requirements**: SRC-01, SRC-02, SRC-03, SRC-04
**Plans**: 4 plans

Plans:
- [x] 22-01: Source coverage API + types + hooks
- [x] 22-02: Source dashboard UI + /sources page
- [x] 22-03: Statement detail + re-import wizard
- [x] 22-04: Incomplete batch banner

**Details:**
- Accordion-based source list with lazy loading

### Phase 23: AI Suggestions & Pattern Detection

**Goal**: System proactively suggests subscriptions based on recurring patterns in statement data
**Depends on**: Phase 22
**Requirements**: AI-01, AI-02, AI-03, AI-04
**Plans**: 4 plans

Plans:
- [x] 23-01: Bulk API + suggestion UI components (ConfidenceBadge, EvidenceList, SuggestionTimeline)
- [x] 23-02: Bulk hooks + BulkActionsBar component
- [x] 23-03: Dedicated /suggestions page + SuggestionCard + sidebar link
- [x] 23-04: Import auto-tagging integration + detection trigger + toast notification

**Details:**
- Pattern detection on recurring charges (3+ occurrences, monthly frequency)
- Bulk FOR UPDATE locking to prevent race conditions
- Fire-and-forget pattern detection trigger
- Auto-tagging of high-confidence items (>80%) during import
- Toast notification with "View Suggestions" action

## Milestone Summary

**Key Decisions:**

- Sequential PDF processing: Prevents memory exhaustion with 50-100MB files
- Keyset pagination: O(1) performance at any depth (vs O(n) OFFSET)
- TanStack Virtual: Lightweight virtualization for 10k+ items
- Many-to-many tags: Flexible tagging system with junction table
- Fire-and-forget detection: Non-blocking pattern detection after batch

**Issues Resolved:**

- Queue processing bug in batch upload (queueRef pattern)
- Stale closure in bulk tag callback (selectedIdsRef pattern)
- Missing database tables (migration applied)
- Batch sources not appearing in import dropdown (cache invalidation)

**Tech Debt:**

- TODO: Blob storage for PDF persistence (currently in-memory)
- TODO: Parser improvements for edge cases
- EvidenceList URL params don't pre-filter (minor UX)
