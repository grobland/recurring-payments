Create a Playwright E2E test that:

1. **Setup: Create test subscription with renewal tomorrow**
   - Navigate to /subscriptions/new
   - Create subscription with unique name like `Reminder Test ${Date.now()}`
   - Amount: $9.99, frequency: monthly
   - IMPORTANT: Set nextRenewalDate to TOMORROW (within the 1-day reminder window)
   - Use date picker to select tomorrow's date OR use direct API call to create subscription with correct date

2. **Trigger cron endpoint via API request**
   - Use Playwright's `request` context (not page) for API call
   - POST to `/api/cron/send-reminders`
   - Include header: `Authorization: Bearer ${process.env.CRON_SECRET}`
   - Assert response is 200 and body contains `success: true`

3. **Verify reminder was sent**
   - Response should include `remindersSent: 1` (or greater)
   - Response should NOT have errors array populated

4. **Test requirements:**
   - Load CRON_SECRET from process.env (dotenv loads .env.local in playwright.config.ts)
   - Skip test gracefully if CRON_SECRET is not configured
   - Add test.describe block: "Email Reminders"
   - Use Playwright's APIRequestContext for the API call

5. **Important considerations from research:**
   - User must have `emailRemindersEnabled: true` (default is true for new users)
   - Subscription must have `reminderEnabled: true` (default is true)
   - Subscription must have `status: 'active'` (default)
   - Subscription must not have been reminded today (deduplication check)
   - User must be on active trial or paid plan (test user should be on trial)

6. **Date picker interaction:**
   The subscription form has a calendar popup. To set tomorrow's date:
   - Click on the date input field to open calendar
   - Find and click tomorrow's date in the calendar
   - OR: Skip date picker, create subscription, then update via API PATCH to set nextRenewalDate

   Simpler approach: Create subscription via API directly using fetch, which gives precise control over nextRenewalDate.

Example test structure:
```typescript
import { test, expect } from "@playwright/test";
import { addDays, format } from "date-fns";

test.describe("Email Reminders", () => {
  test("can trigger email reminder via cron endpoint", async ({ page, request }) => {
    const cronSecret = process.env.CRON_SECRET;
    if (!cronSecret) {
      test.skip(true, "CRON_SECRET not configured");
      return;
    }

    // Create subscription via UI (or API) with renewal date tomorrow
    const tomorrow = addDays(new Date(), 1);
    // ... create subscription ...

    // Trigger cron
    const response = await request.post("/api/cron/send-reminders", {
      headers: { Authorization: `Bearer ${cronSecret}` }
    });

    expect(response.ok()).toBeTruthy();
    const result = await response.json();
    expect(result.success).toBe(true);
    expect(result.remindersSent).toBeGreaterThanOrEqual(1);
  });
});
```
