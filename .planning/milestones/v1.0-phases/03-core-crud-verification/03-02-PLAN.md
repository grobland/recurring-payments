**Required Tests:**

1. **can add a new subscription** (happy path)
   - Navigate to `/subscriptions/new`
   - Fill form fields:
     - Name: use unique name like `Test Sub ${Date.now()}`
     - Amount: `15.99`
     - Currency: select `USD` (form uses combobox with label "Currency")
     - Frequency: select `Monthly` (form uses combobox with label "Frequency")
     - Next Renewal Date: click the date picker button (has CalendarIcon), select a future date
   - Wait for API response with `waitForResponse('**/api/subscriptions')`
   - Click submit button (text: "Create Subscription")
   - Verify toast: "Subscription created successfully"
   - Verify redirect to `/subscriptions`
   - Verify subscription name appears in list

2. **shows validation error for empty name**
   - Navigate to `/subscriptions/new`
   - Click submit without filling name
   - Verify error message: "Name is required" (from Zod schema)

3. **shows validation error for invalid amount**
   - Navigate to `/subscriptions/new`
   - Fill name but set amount to `0` or negative
   - Click submit
   - Verify error message: "Amount must be positive"

4. **can edit an existing subscription**
   - First create a subscription via the UI (or use one created in previous test)
   - Navigate to `/subscriptions`
   - Find the subscription row, click the actions menu (MoreHorizontal button)
   - Click "Edit" menu item
   - Change the amount to a new value
   - Wait for API response with `waitForResponse('**/api/subscriptions/**')`
   - Click "Save Changes" button
   - Verify toast: "Subscription updated successfully"
   - Verify redirect to subscription detail page

5. **can delete a subscription**
   - Create a subscription first
   - Navigate to `/subscriptions`
   - Find the subscription row, click actions menu
   - Click "Delete" menu item
   - Wait for DELETE API response
   - Verify toast shows "deleted" with Undo option
   - Verify subscription no longer appears in list

6. **handles special characters in subscription name**
   - Navigate to `/subscriptions/new`
   - Fill name with special characters: `Netflix Premium HD` (or similar unicode)
   - Complete form and submit
   - Verify subscription is created and displays correctly

**Implementation Notes:**
- Use unique names with `Date.now()` to avoid test isolation issues
- Use `getByLabel()` for form fields where labels exist
- Use `getByRole()` for buttons and menu items
- For date picker, click the button that has "Pick a date" placeholder text
- Use `waitForResponse()` before asserting UI changes
- Use `expect.soft()` for multiple assertions in same test where appropriate
