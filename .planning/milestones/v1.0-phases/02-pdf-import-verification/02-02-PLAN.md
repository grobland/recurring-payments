<task type="auto">
  <name>Task 2: Create E2E test for import flow</name>
  <files>tests/e2e/pdf-import.spec.ts</files>
  <action>
Create a Playwright E2E test that verifies the PDF import flow. The test should:

1. Navigate to the import page
2. Upload a test file using `setInputFiles`
3. Click process and wait for AI processing (with 90s timeout)
4. Verify the review step appears (either with subscriptions or "no subscriptions detected")
5. Handle both success and zero-results paths gracefully

Key patterns from RESEARCH.md:
- Use `path.join(__dirname, '..', 'fixtures', 'filename')` for reliable paths
- Set 90s timeout for AI processing: `{ timeout: 90000 }`
- Use `setInputFiles` for file upload
- Test should pass even if GPT-4o returns zero subscriptions (we're testing the flow, not accuracy)

Test structure:
```typescript
import { test, expect } from '@playwright/test';
import path from 'path';

test.describe('PDF Import Flow', () => {
  test('can upload file and see processing result', async ({ page }) => {
    await page.goto('/import');

    // Upload file
    const filePath = path.join(__dirname, '..', 'fixtures', 'bank-statement-sample.png');
    await page.locator('input[type="file"]').setInputFiles(filePath);

    // Process
    await page.getByRole('button', { name: 'Process Files' }).click();

    // Wait for processing (either success or zero results)
    await expect(page.getByText('Analyzing your documents')).toBeVisible();

    // Wait for review step (generous timeout for AI)
    await expect(
      page.getByText('Review Detected Subscriptions').or(
        page.getByText('No subscriptions detected')
      )
    ).toBeVisible({ timeout: 90000 });
  });
});
```

Handle auth: If the import page requires authentication, the test may need to:
- Use a test account with stored auth state
- Or skip auth for now and add note about future auth setup

For now, assume the test runs against local dev server where auth may be bypassed or a session exists.
