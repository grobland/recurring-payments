## v2.0 Requirements

Requirements for Statement Hub milestone. Each maps to roadmap phases.

### Batch Import

- [x] **BATCH-01**: User can drag-and-drop multiple PDF files (12+) at once
- [x] **BATCH-02**: User sees upload progress indicator per file
- [x] **BATCH-03**: System processes PDFs sequentially to prevent memory exhaustion
- [x] **BATCH-04**: User can resume interrupted batch (per-file status tracking)
- [x] **BATCH-05**: System detects duplicate statements and prompts user to skip or re-import

### Statement Data

- [x] **DATA-01**: System stores ALL line items from statements (not just subscriptions)
- [x] **DATA-02**: Each line item links to its source (bank/credit card name)
- [x] **DATA-03**: System fingerprints transactions for deduplication (merchant + amount + date)
- [x] **DATA-04**: System auto-tags high-confidence items (>80%) as potential subscriptions

### Statement Browser

- [x] **BRWS-01**: User can view list of all transactions across all sources
- [x] **BRWS-02**: User can filter transactions by source (bank/card)
- [x] **BRWS-03**: User can filter transactions by date range
- [x] **BRWS-04**: User can search transactions by merchant name
- [x] **BRWS-05**: Browser uses virtualized scrolling for 10k+ items
- [x] **BRWS-06**: User can filter transactions by tags (potential, converted, dismissed)

### Manual Enrichment

- [x] **ENRCH-01**: User can tag any transaction as "potential subscription"
- [x] **ENRCH-02**: User can convert any transaction to subscription with one click
- [x] **ENRCH-03**: User can bulk-tag multiple transactions at once
- [x] **ENRCH-04**: Subscription form pre-fills with transaction data (name, amount, date)

### Source Management

- [x] **SRC-01**: User sees source dashboard with cards per bank/credit card
- [x] **SRC-02**: Each source card shows coverage dates (earliest to latest statement)
- [x] **SRC-03**: User can view statement detail (all items from a specific import)
- [x] **SRC-04**: User can re-import skipped items from previous statements

### AI Suggestions

- [x] **AI-01**: System detects recurring patterns in statement data
- [x] **AI-02**: System suggests subscriptions with evidence (occurrences, dates, amounts)
- [x] **AI-03**: User can accept suggestion (creates subscription)
- [x] **AI-04**: User can dismiss suggestion (won't show again)

## Future Requirements (Deferred)

### Production

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Automatic reconciliation | Enterprise complexity users don't need |
| Transaction splitting | Rare use case, adds UX friction |
| Budgeting features | Different product domain |
| Bank API integration (Plaid) | Security/compliance burden, PDF import sufficient |
| Mobile app | Web-first approach |
| OpenAI Batch API | 24-hour latency unacceptable for import flow |
| Parallel PDF processing | Memory exhaustion risk |
| OFFSET pagination | Performance collapse at scale |
