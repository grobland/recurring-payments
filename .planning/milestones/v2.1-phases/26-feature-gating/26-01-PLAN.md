Create `src/lib/features/server.ts` with `"use server"` directive:

1. **hasFeature(feature: Feature): Promise<boolean>**
   - Get session via `auth()` from `@/lib/auth`
   - Return false if not authenticated
   - Get user tier via `getUserTier(session.user.id)` from `@/lib/stripe/tiers`
   - Return `canTierAccessFeature(userTier, feature)`

2. **getUserFeatureAccess(feature: Feature): Promise<{ hasAccess: boolean; userTier: Tier | null; requiredTier: Tier }>**
   - Get session via `auth()`
   - If not authenticated, return { hasAccess: false, userTier: null, requiredTier: getRequiredTier(feature) }
   - Get user tier and check access
   - Return full access info object

3. **requireFeature(feature: Feature): Promise<void>**
   - Call hasFeature()
   - If false, throw Error with message: `This feature requires ${requiredTier} tier`
   - Error can be caught by API routes to return 403

Pattern follows: Existing server actions in tiers.ts
