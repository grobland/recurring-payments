   - Auth check + feature gate with isUserActive
   - Query params: status (active/inactive/all, default "active"), minConfidence (optional decimal), cursor (cursorDate + cursorId), pageSize (default 20, max 50)
   - Query: `db.select({...seriesFields, merchantName: merchantEntities.name}).from(recurringSeries).leftJoin(merchantEntities, eq(recurringSeries.merchantEntityId, merchantEntities.id)).where(and(eq(recurringSeries.userId, session.user.id), ...filters)).orderBy(desc(recurringSeries.updatedAt), desc(recurringSeries.id)).limit(pageSize + 1)`
   - Cursor pagination using (updatedAt, id) compound cursor
   - Response: `{ data: [...], pagination: { hasMore, nextCursor } }`

2. Create `src/app/api/recurring/series/[id]/route.ts` — GET (detail):
   - Auth check
   - Fetch series by id with userId scope
   - Fetch linked transactions: `db.select({...txFields, matchConfidence: recurringSeriesTransactions.matchConfidence}).from(recurringSeriesTransactions).innerJoin(transactions, eq(recurringSeriesTransactions.transactionId, transactions.id)).where(eq(recurringSeriesTransactions.seriesId, id)).orderBy(desc(transactions.transactionDate))`
   - Fetch merchant entity name if merchantEntityId present
   - Check if linked to a master via recurringMasterSeriesLinks
   - Return: `{ data: { ...series, merchantName, linkedMaster: { id, name } | null, transactions: [...] } }`

3. Create `src/app/api/recurring/series/[id]/confirm/route.ts` — POST:
   - Auth check + feature gate
   - Parse body with confirmSeriesSchema
   - Verify series exists, belongs to user, is active
   - If `existingMasterId` provided: verify master exists and belongs to user, then link series to master via recurringMasterSeriesLinks insert, log recurringEvent type "linked"
   - If no existingMasterId: create new recurringMaster with name, recurringKind from body, derive currency/amount/frequency from series fields (avgAmount, detectedFrequency, dayOfMonth, currency), set status "active", confidence from series. Insert recurringMasterSeriesLinks with isPrimary=true. Log recurringEvent type "created".
   - All writes in single db.transaction()
   - Return: `{ success: true, masterId: string, action: "created" | "linked" }`

4. Create `src/app/api/recurring/series/[id]/ignore/route.ts` — POST:
   - Auth check + feature gate
   - Verify series exists and belongs to user
   - Update series: `db.update(recurringSeries).set({ isActive: false, updatedAt: new Date() }).where(and(eq(recurringSeries.id, id), eq(recurringSeries.userId, session.user.id)))`
   - If series is linked to a review queue item, resolve it as "ignored": `db.update(reviewQueueItems).set({ resolvedAt: new Date(), resolution: "ignored" }).where(and(eq(reviewQueueItems.seriesId, id), isNull(reviewQueueItems.resolvedAt)))`
   - Return: `{ success: true }`
