# Milestone v1.3: Data & Intelligence

**Status:** SHIPPED 2026-02-08
**Phases:** 13-18
**Total Plans:** 21

## Overview

Transform raw subscription data into actionable insights with duplicate detection, pattern recognition, and comprehensive spending analytics.

## Phases

### Phase 13: Analytics Infrastructure

**Goal:** Analytics foundation with pre-computed aggregates for all intelligence features
**Depends on:** Phase 12
**Plans:** 3 plans

Plans:
- [x] 13-01: Database materialized view + cron refresh infrastructure
- [x] 13-02: Analytics API endpoint + TanStack Query hook
- [x] 13-03: Dashboard UI (period selector, stat cards, donut chart)

**Details:**
- Materialized view `user_analytics_mv` with 15-minute cron refresh
- Currency normalization to user's default currency
- Dashboard shows monthly/yearly totals and category breakdown

### Phase 14: Duplicate Detection

**Goal:** Users are warned about potential duplicates during import and can find duplicates in existing subscriptions
**Depends on:** Phase 13
**Plans:** 4 plans

Plans:
- [x] 14-01: TDD: Multi-field weighted similarity algorithm
- [x] 14-02: Import-time duplicate detection with warnings and actions
- [x] 14-03: Background scan, merge API, and subscriptions page UI
- [x] 14-04: Integration wiring and verification checkpoint

**Details:**
- Jaro-Winkler similarity with 70% detection threshold
- Keep/Skip/Merge actions during import
- Field-by-field merge picker with 24-hour undo
- Background scan finds duplicates in existing subscriptions

### Phase 15: Spending Analytics & Trends

**Goal:** Users can visualize spending trends over time and understand how spending changes
**Depends on:** Phase 13
**Plans:** 3 plans

Plans:
- [x] 15-01: Trends API endpoint + useTrends hook
- [x] 15-02: TrendIndicator component + analytics cards enhancement
- [x] 15-03: YoY chart + category trends chart + dashboard integration

**Details:**
- Month-over-month indicators with up/down arrows
- Year-over-year dual-line comparison chart
- Per-category trend lines with database colors

### Phase 16: Pattern Recognition

**Goal:** Users receive suggestions for subscriptions based on recurring charges detected across statements
**Depends on:** Phase 13
**Plans:** 3 plans

Plans:
- [x] 16-01: Schema + pattern detection algorithm + category guesser
- [x] 16-02: API endpoints (detect, suggestions, accept, dismiss) + hooks
- [x] 16-03: Dashboard UI (PatternSuggestionsCard) + integration

**Details:**
- PostgreSQL LAG window function for interval detection
- Multi-factor confidence scoring (occurrence 30%, interval 40%, amount 30%)
- Keyword-based category guessing for 14 categories
- Accept/dismiss with undo support

### Phase 17: Spending Forecasting

**Goal:** Users can view predicted future spending with confidence intervals
**Depends on:** Phase 15
**Plans:** 4 plans

Plans:
- [x] 17-01: Types, utilities, and calendar API endpoint
- [x] 17-02: Monthly and annual API endpoints + useForecast hooks
- [x] 17-03: Calendar view component with day selector
- [x] 17-04: Forecast charts + dashboard integration + verification

**Details:**
- Upcoming charges calendar with 30/60/90 day range
- Monthly projections with 80% and 95% confidence intervals
- Fan chart visualization with sqrt(time) uncertainty scaling
- Annual forecast extrapolated from current subscriptions

### Phase 18: Anomaly Detection & Alerts

**Goal:** Users are alerted to unusual spending patterns and subscription changes
**Depends on:** Phase 15
**Plans:** 4 plans

Plans:
- [x] 18-01: Database schema + detection utilities + cron job
- [x] 18-02: Alerts API endpoints + TanStack Query hooks + price detection
- [x] 18-03: Notification bell UI + dropdown component
- [x] 18-04: Weekly digest email + cron job + verification

**Details:**
- Price increase detection (>5% or >$2)
- Missed renewal detection (3+ days overdue)
- Notification bell with red indicator in header
- Weekly digest email batching (Monday 8am UTC)

## Milestone Summary

**Key Decisions:**
- Materialized view pattern for fast analytics queries (<100ms)
- 70% similarity threshold for duplicate detection
- sqrt(time) scaling for forecast uncertainty
- Weekly batch over real-time alerts to prevent fatigue
- Fire-and-forget pattern triggers for async operations

**Issues Resolved:**
- Materialized view UNIQUE INDEX required for CONCURRENT refresh
- PostgreSQL LAG window function for time-series analysis
- Stacked area pattern for Recharts fan chart visualization

**Technical Debt Incurred:**
- Hooks not re-exported from central index (use-duplicate-scan, use-merge-subscription, useTrends, useForecast*, useAlerts, etc.)
- Phase 14 verification inline in summary rather than separate file
